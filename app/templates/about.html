About Poster Management System

System Overview

The Poster Management System is a web-based application designed to streamline the poster production workflow for the Party for Socialism and Liberation. It replaces manual Python scripts with an intuitive web interface accessible via Docker container on a Tailscale network.

Key Features
  • Web-Based Interface: Accessible from any device on the Tailscale network
  • Role-Based Authentication: Viewer, Contributor, and Admin roles
  • Automated Bug Generation: Creates standardized bug pages with QR codes and metadata
  • Orientation Support: Automatic rotation for landscape posters
  • Kit & Collection Management: Organize posters into physical kits and binder collections
  • Inventory Tracking: Track counted and printed posters per collection
  • Printable Labels: Generate 8.5×11" kit labels and 11×17" collection labels
  • FTP Export: Automatic export of print-ready PDFs to printer FTP directory
  • File-Based Storage: All configuration and metadata stored in human-readable YAML/JSON files
  • Weekly Backups: Automatic ZIP backups with configurable retention
Technical Architecture

The system is built with Flask (Python) using server-rendered Jinja2 templates. All data is stored in human-readable files (YAML for configuration, JSON for metadata) to facilitate easy backup, version control, and manual editing when needed.

The application runs in a Docker container with persistent volume mounts for configuration, data storage, and backups. Authentication uses bcrypt password hashing with session-based access control.

Security Notes
Important

This system is designed for internal use on a secure Tailscale network. Default admin credentials should be changed immediately after first login.

  • All passwords are hashed with bcrypt
  • Session cookies are HTTP-only
  • File uploads are validated and limited to 200MB
  • Configuration files are stored separately from application code
  • Weekly automatic backups to separate volume
